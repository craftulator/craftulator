- Context: Vite had a manual `resolve.alias` map that duplicated path aliases already defined in `tsconfig.json`.
- Decision: Add `vite-tsconfig-paths` as an exact dev dependency and use it in `vite.config.ts` instead of maintaining a
  duplicate manual alias map.
- Consequences: Vite and Vitest now resolve project aliases from `tsconfig.json`, reducing config drift when aliases
  change.
- Status: accepted
