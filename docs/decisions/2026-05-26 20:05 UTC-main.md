- Context: Craftulator needs a documented implementation order before site implementation starts.
- Decision: Track the site implementation in `docs/roadmap.md` with milestones and checkboxes. Start with project
  configuration and branding, then routing, shared components, the home page, incremental editor work, polish, and only
  then CI/CD deployment to GitHub Pages.
- Consequences: Implementation tasks should follow the roadmap order and keep the application buildable after each small
  step. The editor and icon-adding flow remain full-scope features, but they must be delivered incrementally.
- Status: accepted
