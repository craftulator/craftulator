- Context: Milestone 2 requires production favicon assets and theme logo availability before the app shell and page work
  use them.
- Decision: Publish favicon outputs and theme logos under Vite's `public/` directory so they are copied unchanged to the
  build output and can be referenced by root-relative URLs.
- Consequences: `index.html` can use stable favicon and manifest paths immediately, and future theme work can reference
  `/theme-logos/<theme>.png` without adding bundler imports yet.
- Status: accepted
