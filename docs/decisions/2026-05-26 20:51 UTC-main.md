- Context: BrowserRouter needs direct-route support on GitHub Pages, but GitHub Pages serves `404.html` for unknown deep
  links.
- Decision: Use a small `public/404.html` redirect shim that stores the requested path in `sessionStorage`, redirects to
  `/`, and lets `index.html` restore the original path before the app mounts.
- Consequences: Direct links such as `/editor` can load the SPA on GitHub Pages without adding a router basename. The
  fallback is root-site specific and should be revisited only if the deployment target changes away from
  `https://craftulator.github.io/`.
- Status: accepted
