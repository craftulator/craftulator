- Context: The React template needs stronger conventions for architecture, imports, i18n, tests, and maintainable
  commits.
- Decision: Adopt FSD-like layers, keep `src/pages` as URL-shaped re-exports from `@templates`, enforce separate
  type-only imports, use `useIntl()` with `defineMessages()`, colocate non-e2e tests, and document small logical
  commits.
- Consequences: The project has clearer boundaries, route components are imported through `@pages`, page code lives in
  templates, and future contributors have explicit style and structure rules.
- Status: accepted
